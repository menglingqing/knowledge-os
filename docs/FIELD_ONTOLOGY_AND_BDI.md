## 一、BDI 模型回顾

**Belief-Desire-Intention**，经典智能体架构。

```
Belief（信念）  →  我对世界的认知是什么？
Desire（愿望）  →  我想要达成什么？
Intention（意图）→  我现在决定做什么？
```

```
环境 → Belief → Desire → Intention → Action
线性因果链，适合写 Agent 代码。
```

---

## 二、BDI 的致命漏洞

**解释不了「欲望从哪里来」。**

同一个沙漠环境：
- 有的人想找水
- 有的人想自拍
- 有的人想写诗

BDI 说 Desire 来自 Belief + 环境刺激。但它解释不了：同一个环境，同一个 Belief，为什么 Desire 不同？

**因为光照的方向不同。** 而 BDI 没有「谁决定看什么」这个维度。

---

## 三、场域模型 vs BDI

```
BDI 模型（行为层）：
  环境 → 信念 → 欲望 → 意图 → 行动
  站在外部看人，刺激-反应链条

场域模型（存在层）：

       光（目标）
        │
        ▼
  ┌─────────────────────┐
  │      场域            │
  │  ┌───────────┐      │
  │  │ 被照亮的部分 │◄─ 人（筛选器/权重）
  │  └───────────┘      │
  │       ↑             │
  │  环境 = 可观测边界   │
  └─────────────────────┘
```

| 要素 | BDI 中的角色 | 场域模型中的角色 |
|------|-------------|-----------------|
| 需求 | 环境刺激产物 | 场域被光照亮后才构成的 |
| 目标 | 需求驱动的结果 | 光本身——决定什么被看见 |
| 环境 | 客观给定的输入 | 场域被照亮的可观测边界 |
| 人 | 被动反应者 | 起心动念者，光照方向的来源 |

---

## 四、场景与场域的关系

### 最终定义

> 场域是本体，不随人变。人起心动念时目标激活、光打进场域，显现出一个范围。环境控制这个范围的大小和形状。人在这个被光照亮又被边界围住的区域里，用自己的偏好给每一条路加权。

### 场景 = 场域的显现

```
场域 ──显现为──→ 场景

不是「场域包含场景」（切片模型），
而是「场域显现为场景」（显现模型）。

场域 = 场景的这个「在」
场景 = 场域的这个「显」
```

### 两步模型

```
第一步：场域 → 显现为 → 场景（不依赖人，一直在发生）
第二步：人进入场景 → 起心动念（光）→ 环境被裁切 → 场景被重新结构化

第一层是存在论，第二层是认知论。
两件不同的事。
```

## 五、四要素关系

| 要素 | 角色 | 比喻 | 稳定性 |
|------|------|------|--------|
| 场域 | **本体**——一直在那 | 黑暗中的整张地图 | 极高 |
| 目标 | **显现**——起心动念的瞬间 | 手电筒的光 | 低（一天可变多次） |
| 环境 | **边界**——控制显现范围 | 光束角和亮度 | 中 |
| 人 | **加权**——在显现区域内排序 | 站在光圈里选路 | 高 |

## 六、映射到编程

```
场域   → 整个代码库的可能性空间（所有能做的事）
光     → 架构意图（为什么要建这段代码）
环境   → 类型系统、接口约束（你能做什么、不能做什么）
筛选器 → Selector（在多个策略里选哪个执行）
```

场域模型比 BDI 更适合描述**开放式工程决策**——真正的工程问题，需求和目标常常是一起浮现的，而不是先有需求再有目标。
