# 人工替代率与增强效率

> **知识类型**: concept

---

## 一、核心误解 vs 正解

```
误解：人工替代率 80% = 裁掉 80% 的人
正解：人工替代率 80% = 原来 100% 的工作需要人做，现在 80% 由系统处理，
      人只做剩下的 20%——但人做的那 20% 是决策、异常、创造。
```

**替代的不是人，替代的是动作。**

---

## 二、增强效率的三层含义

### 第一层：时间置换

```
原来：1 个客服 8 小时处理 80 个咨询
现在：1 个客服 8 小时处理 80 个复杂咨询 + AI 处理 320 个简单咨询
总产能 80 → 400。人没变多，人的时间花在更有价值的事上。
```

### 第二层：能力放大

```
原来：客服能记住 20 个常见问题的标准答案
现在：AI 实时推送答案 + 查历史 + 预测用户情绪，
     客服借助 AI 能处理的知识面翻了 10 倍。
人没变，决策半径变大了。
```

### 第三层：结构优化

```
原来：招人 → 培训 → 上岗，3 个月。离职 → 重新来。
现在：AI 吃掉标准化判断，新人借助 AI 3 天能顶老员工 80% 的活。
组织对「人」这个变量的依赖在降低。
```

## 三、人工替代率打穿预算结构

```
假设：ARR 7000 万，1000 个客户，3.6 万次支持请求/年

人工替代率 0%：
  3.6 万次 ÷ 1500 次/人/年 = 24 个客服
  人力成本：24 × 15 万/年 = 360 万

人工替代率 60%：
  人处理：1.44 万次（复杂的）
  AI 处理：2.16 万次（标准化的）
  人力：1.44 万 ÷ 1500 = 9.6 人 → 10 人
  + AI 训练/运营 2 人（新角色）
  人力成本：10 × 15 万 + 2 × 25 万 = 200 万
```

**结果不是总成本减少，是结构的重新分配：**

```
替代前         替代后
───────        ───────
客服 55%      客服 25%
              AI 运营 15%  ← 新角色，新预算条目
               技术 15%    ← 工具采购、API 成本
管理 10%      管理 8%
```

---

## 四、一句话

> 人工替代率衡量的是「多少判断从人脑移交给了系统」，不是「多少人丢了工作」。
> 增强效率 = 同样的人做出更大的决策半径 + 人的时间集中在系统做不了的事上。
> 在预算表上，它不减少总成本，它重新分配结构。
