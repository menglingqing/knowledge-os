# NDR 净收入留存

> **日期**: 2026-06-16
> **来源**: 老板培训 — 经营预算核心指标
> **知识类型**: concept
> **标签**: #NDR #ARR #SaaS #留存 #经营指标

---

## 一、一句话定义

**NDR（Net Dollar Retention）= 同一批老客户，今年给你的钱是去年的百分之多少。**

```
NDR = (Renewal + Expansion - Churn) / 上期同批客户的 ARR
```

只看老客户。不看新客。回答：老客户在长大还是在流失？

---

## 二、与 ARR 的核心区别

```
ARR 回答：总盘子多大？
NDR 回答：老客户的质量如何？
```

### 为什么 ARR 会骗人，NDR 不会

```
版本 A：ARR 10M → 15M，增长 50%，牛逼。
  拆开：新客贡献 $8M，NDR 70%（老客户在流失）
  诊断：水桶漏水。获客成本上升时，增长悬崖。

版本 B：ARR 10M → 13M，增长 30%，一般。
  拆开：新客贡献 $2M，NDR 110%（老客户在升级）
  诊断：飞轮在转。产品好到客户越用越深。
```

---

## 三、NDR 五因子

```
NDR = 100% 起点
       + 升级（Expansion）   客户从 Pro 升 Enterprise
       + 加购（Upsell）      在现有套餐上加功能模块
       + 扩量（Volume）      API 调用量翻了
       - 降级（Downgrade）   Enterprise 退回 Pro
       - 流失（Churn）       不续了
```

---

## 四、健康度分级

| NDR | 评级 | 含义 |
|-----|------|------|
| > 120% | 顶级 | Snowflake、CrowdStrike 级别 |
| 110-120% | 优秀 | 产品自己卖 |
| 100-110% | 合格 | 守住，靠新客增长 |
| < 100% | 漏水 | 老客在萎缩 |
| < 80% | 漏斗 | 获客追不上流失，死亡螺旋 |

**大多数 AI 公司 NDR < 100%。** Individual 用户不升级、不续费、下个月换竞品。

---

## 五、NDR 决定预算结构

```
NDR > 100%：预算重研发、轻销售。产品自己卖。
  研发 35%，销售 10%

NDR < 100%：预算重销售。必须不断买新客填漏水的桶。
  研发 10%，销售 35%
```

同一个 ARR 目标，NDR 不同，预算结构完全不同。**NDR 是结构决策的前提。**
