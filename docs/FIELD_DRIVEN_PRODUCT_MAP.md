# 场域驱动 · 爆款图谱认知框架

> **日期**: 2026-06-15
> **起点**: Xmoom PLM 需求分析
> **方法**: 苏格拉底追问 + 第一性原理拆解（乔布斯/马斯克式）
> **产出**: 一套从"商品驱动"到"场域驱动"的完整认知跃迁

---

## 一、核心认知跃迁

### 1.1 根基洞察

> **"衣服从来不是商品，那一天的我才是。"**

- 消费者买的不是一件衣服，是"那天要发生的事"
- 风格定义 = 从商品特征出发（供应链语言）
- 场景分析 = 从人的角度出发（消费者决策语言）

### 1.2 范式对比

| 维度 | 风格驱动（旧范式） | 场域驱动（新范式） |
| --- | --- | --- |
| 思维起点 | 这件衣服长什么样 | 穿衣服的人要过什么样的日子 |
| 组织语言 | 供应链语言（面料、风格、品类） | 消费者语言（场景、目标、决策瞬间） |
| 风险兑现 | 上架后等数据 | 画图前在假设层验证 |
| 设计师能力 | 全能型——什么风格都能画 | 专注型——吃透一条场景线的特定客群 |
| 做减法的方式 | 主观审美 | 场景约束自动排除 |
| 商品角色 | 自变量 | 因变量——场域的填充方案 |
| 人的需求 | 被商品说明书化 | 稳定不变的结构 |

## 二、场域本体论

### 2.1 最终定义

> **场域是本体，不随人变。人起心动念时目标激活、光打进场域，显现出一个范围。环境控制这个范围的大小和形状。人在这个被光照亮又被边界围住的区域里，用自己的偏好给每一条路加权——最后走哪条，是人的选择。场域从不消失，只是你不看它的时候，它是暗的。**

### 2.2 四要素关系

| 要素 | 角色 | 比喻 | 稳定性 |
| --- | --- | --- | --- |
| 场域 | **本体**——一直在那 | 黑暗中的整张地图 | 极高（需求空间客观存在） |
| 目标 | **显现**——起心动念的瞬间 | 手电筒，照向某个方向 | 低（一天可变多次） |
| 环境 | **边界**——控制显现范围 | 手电筒的光束角和亮度 | 中（气候周期、物理规律不变） |
| 人 | **加权**——在显现区域内排序 | 站在光圈里看哪条路适合自己 | 高（身体和身份以年为单位变化） |

### 2.3 关键修正

初始理解：三者交汇产生场域。

**修正后（用户提出）**：场域一直存在。不是"三者交汇产生场域"，而是"人起心动念时，目标的光打进场域，显现出被环境划定边界的一块区域——人在这块区域里加权筛选"。

这个修正回答了一个关键问题：**为什么不同品牌可以不约而同发现同一个需求空间？**——因为它们没有创造场域，只是先后感知到了同一个早已存在的需求空间。

## 三、场域运算模型

```
场域（本体，一直存在）
   ↓ 目标激活（起心动念）
显现区域
   ↓ 环境边界（物理+社会+时间约束）
约束条件集
   ↓ 人加权筛选（偏好+身体+身份）
优先级排序
   ↓
[唯一的生存区间]
```

### 实例推演

```
人：小杨，32，梨形，杭州，市场经理
目标：重要项目汇报，显得专业但不刻板
环境：秋季 12-22°C / 会议室冷白光 / 全天 / 晚有饭局
        ↓ 光打进场域
      【场域显现区域】
        ↓ 场域产出
约束 1（环境）：厚度抗空调 + 久坐不皱 → 权重 10
约束 2（目标）：专业但不刻板 → 权重 9
约束 3（人）：遮大腿 + 收腰 → 权重 9
约束 4（环境）：衬冷白光不暗沉 → 权重 7
约束 5（目标）：见朋友不用换 → 权重 6
        ↓ 约束交集
   [不在这个区间内的商品，在这个场域里卖不动]
   [不是不好看，是不成立]
```

                    │  │ └────────────────────────┘ │  │

### 4.2 五层定义

| 层 | 颜色 | 内容 | 回答的问题 | 稳定性 |
| --- | --- | --- | --- | --- |
| ① 场域中心 | 琥珀光晕 | Persona × Goal × Environment 交叉点，呈现为"问题陈述"而非"商品图片"。抽象符号：三圆交叠汇于一点 | "问题在哪里？" | 极高 |
| ② 约束环 | 青碧 | 物理约束（温度/体感/活动量）+ 社会约束（正式度/辨识度/年龄感）+ 身体约束（遮蔽区/展示区/版型铁律） | "什么条件必须同时成立？" | 高 |
| ③ 候选基因环 | 琥珀 | 约束→属性的逻辑翻译：面料方案、版型方案、色域方案、设计元素，每项带可行性得分 | "哪些属性组合在逻辑上可行？" | 中 |
| ④ 商品证据环 | 玫瑰金 | 竞品验证 + 自有验证 + 商品家族基因交集徽章。全屏唯一出现商品图像的位置（小邮票级、去模特去背景、纯商品证件照） | "哪些组合已经被市场验证过？" | 中低 |
| ⑤ 信号环 | 紫罗兰 | 场域满足度 + 最大失分点 + 竞品截获流向 + 需求空隙决策优先级 | "我们在这个场域的位置？该不该打？" | 低 |

### 4.3 核心设计决策

**中心为什么不是商品图片？**
因为要找的东西还不存在。旧图谱解释"这件为什么成功"，新图谱定位"哪里缺一件"。

**唯一出现商品图的是第四环。为什么？**
因为它展示的不是可能性——是已经发生的事实。候选基因环说"应该可行"，证据环说"已经被验证了——或者还没人做到，这就是你的空隙"。

**第四环为什么是玫瑰金色？**
琥珀（候选基因环）= 逻辑推理，可能性。玫瑰金（商品证据环）= 已经发生的事实，触摸得到的真相。颜色区分信息类型。

**"商品家族基因交集"为什么是六边形徽章？**
单个爆款可能是偶然。三件不同品牌的成功商品共享同一组基因——那是这个场域的底层生存法则。六边形暗示"结构、规律、确定性"。

### 4.4 旧图谱 vs 新图谱

| | 旧图谱 | 新图谱 |
| --- | --- | --- |
| **中心** | 一件具体商品（答案） | 一个场域（问题） |
| **环的内容** | 商品基因拆解（解释） | 约束条件的交集 + 证据 + 信号 |
| **做的事** | 告诉设计师"这个爆了，像这样" | 告诉企划师"这个地方缺，往这个边界里做" |
| **思维模式** | 跟着商品属性走——被动 | 从人的需求出发——主动 |
| **商品角色** | 自变量 | 因变量 |
| **人的需求** | 被商品说明书化 | 稳定不变的结构 |
| **核心动作** | 事后解释 | **事前推演** |
| **30秒交付** | 一件衣服的基因 | **最值得投入的下一个需求空隙** |

## 五、方法论：苏格拉底追问 × 第一性原理

### 5.1 本次使用的方法

| 阶段 | 方法 | 产出 |
| --- | --- | --- |
| 第一轮 | 从消费者行为最底层开始问 | "她买的是那天发生的事" |
| 第二轮 | 路径 A vs 路径 B 结构化对比 | "场景分析从人的角度，风格定义从商品特征出发" |
| 第三轮 | 第一性原理拆解（马斯克式） | 五个原子事实 → 五层架构 |
| 第四轮 | 用户体验倒推（乔布斯式） | "30秒定位需求空隙" |
| 第五轮 | 本体论追问 | 场域一直存在，目标是光，环境是边界，人是加权器 |

### 5.2 用户思维特征

| 特征 | 证据 |
| --- | --- |
| 商业直觉精准 | 第一轮即抓住"那天发生的事" |
| 抽象能力 | 三轮追问提炼"人 vs 商品"范式对比 |
| 直觉-框架型思维 | 先有直觉，再用框架显性化 |
| 能自我修正 | 将"场域是三者交汇产生"修正为"场域是本体，一直存在" |
