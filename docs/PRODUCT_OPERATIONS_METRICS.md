## 一、产品运营的核心命题

```
产品做出来了 → 有人用吗？
              → 用对了吗？
              → 用完了还回来吗？
              → 用着用着卡在哪？
```

ARR 和 NDR 回答「收钱」端。产品运营回答「用」端。

---

## 二、四层指标树

### 第一层：有没有人用

| 指标 | 定义 |
|------|------|
| DAU/MAU | 日活/月活 |
| 激活率 | 注册后完成首次关键动作的比例 |
| 新用户留存 | Day 1 / Day 7 / Day 30 |

### 第二层：用得深不深

| 指标 | 定义 |
|------|------|
| 人均使用频次 | 每天/每周打开几次 |
| 人均使用时长 | 每次用多久 |
| 功能渗透率 | 某个功能被多少用户用过 |
| 核心动作完成率 | 用户是否走到了最该走的那一步 |

### 第三层：用得顺不顺

| 指标 | 定义 |
|------|------|
| 任务成功率 | 想完成的事完成了没有 |
| Time to Value | 从打开到获得第一次价值，花了多久 |
| 报错率 / 卡顿率 | 用着用着崩了多少次 |
| 客服 ticket 率 | 每 100 个用户产生多少工单 |

### 第四层：用了还回不回来

| 指标 | 定义 |
|------|------|
| 留存率 | 次留 / 7留 / 30留 |
| 流失率 | 走了多少 |
| 复活率 | 走了又回来多少 |
| NDR | 回来的人花更多还是更少——这里跟财务接上 |

---

## 三、三个北极星指标

| 指标 | 为什么 | 跟预算的关系 |
|------|--------|-------------|
| **激活率** | 第一关。拉新的钱，激活没做起来等于白花 | 决定 CAC 能不能回收 |
| **留存率** | 用了一次还会不会来。留存崩了，ARR 只是新客幻觉 | 决定 NDR 的底座 |
| **核心动作完成率** | 不是「打开了」，是「用成了」 | 决定 Expansion ARR 有没有土壤 |

---

## 四、产品运营与预算的连接

产品运营指标是**服务量的实质**：

```
服务量（表面）：1000 个客户，3.6 万次支持请求，24 个客服

服务量（实质）：激活率 40% → 600 个激活了
               D7 留存 55% → 330 个留住了
               核心动作完成率 30% → 99 个真正用透了
               ↓
              客服不是在服务 1000 个客户，
              是在给 901 个没用起来的人救火。
```

**每 1% 的激活率没做上去，成本会从客服 ticket、客户流失、销售补单三个方向回流，打穿预算结构。**
